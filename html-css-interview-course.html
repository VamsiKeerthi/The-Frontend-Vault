The building blocks every interviewer expects you to know cold

DOCTYPEElementsAttributesHead vs BodyInline vs Block
Q.01 What is the DOCTYPE declaration and why is it needed? +

<!DOCTYPE html> tells the browser which version of HTML the document uses. Without it, browsers enter quirks mode ‚Äî emulating old, buggy behavior from the 1990s that can break your layout.

The HTML5 doctype is the simplest form and triggers standards mode in all modern browsers. It must be the very first thing in the file ‚Äî before the <html> tag.

<!DOCTYPE html>  <!-- HTML5 doctype ‚Äî always use this -->
<html lang="en">
  <head>...</head>
  <body>...</body>
</html>
üí° Interview Tip

Mention quirks mode vs standards mode. Say "Without DOCTYPE, box model sizing and vertical margins can behave unexpectedly across browsers."

Q.02 What's the difference between block-level and inline elements? +
Block-levelInline
Starts on a new lineFlows within text
Takes full available widthOnly as wide as content
Can contain other block/inlineShould not contain block elements
div, p, h1‚Äìh6, ul, section, articlespan, a, strong, em, img, input
/* You can change display type with CSS */
display: block;        /* make inline act as block */

üí° Interview Tip

Bring up inline-block ‚Äî it sits inline but respects width, height, and vertical margins. Great for buttons.

Q.03 What goes inside <head> vs <body>? +

The <head> contains metadata ‚Äî information about the page that isn't displayed to users. The <body> contains the visible content.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Page Title</title>
  <link rel="stylesheet" href="style.css">
  <meta name="description" content="Page description">
  <!-- Open Graph, Twitter cards, canonical, favicon... -->
</head>
Q.04 What is the difference between <script> placed in head vs body? What do defer and async do? +

Script loading blocks HTML parsing. defer and async let the browser download scripts without blocking parsing.

üí° Interview Tip

Use defer for most scripts. Use async for independent scripts like analytics that don't depend on DOM or other scripts.

Q.05 What are void elements in HTML? Give examples. +

Void elements cannot have children and don't need a closing tag. They are self-closing by nature.

<input type="text" name="username">
<br>   <hr>   <meta>   <link>   <source>   <area>

In HTML5, you can write them as <br /> (XHTML style) but the trailing slash is optional and ignored.

Q.07 What is the difference between <strong>/<em> and <b>/<i>? +

<strong> and <em> carry semantic meaning. Screen readers stress them. <b> and <i> are purely presentational.

Q.08 What is the difference between <link>, @import, and inline styles? +

Three ways to apply CSS ‚Äî each with different performance implications.

<!-- 1. <link> ‚Äî RECOMMENDED: loads in parallel -->
<link rel="stylesheet" href="style.css">

<!-- 2. @import ‚Äî loads sequentially, blocks rendering -->
<style>
  @import url('style.css');
</style>

<!-- 3. Inline ‚Äî highest specificity, hardest to maintain -->
<p style="color: red;">Text</p>
‚ö†Ô∏è Performance Note

@import creates a waterfall ‚Äî the browser must download and parse the CSS file before it discovers the imported file. Always prefer <link>.

SEMANTIC HTML

Meaningful markup ‚Äî because context matters to browsers, bots, and screen readers

Q.01 What is semantic HTML and why does it matter? +

Semantic HTML uses elements that describe their meaning, not just their appearance. <article> means "self-contained content." <nav> means "navigation." Contrast with <div> and <span> which carry no meaning.

Benefits: Better SEO (search engines understand page structure), better accessibility (screen readers use landmarks), easier maintainability, and improved collaboration.

<!-- Non-semantic -->
<div class="header"><div class="nav">...</div></div>

<!-- Semantic -->
<header><nav>...</nav></header>
Q.02 Explain the difference between <article>, <section>, and <div>. +
ElementUse when...
<article>Content is self-contained and could stand alone (blog post, news article, forum post, widget). Could be syndicated via RSS.
<section>A thematic grouping of content that belongs together. Should have a heading. Part of a larger whole.
<div>No semantic meaning needed ‚Äî purely for styling/layout grouping.
<article>
  <h2>How to bake bread</h2>
  <section>
    <h3>Ingredients</h3>
    ...
  </section>
  <section>
    <h3>Steps</h3>
    ...
  </section>
</article>
Q.03 When should you use <header>, <main>, <footer>, <aside>, and <nav>? +

These are landmark elements ‚Äî screen reader users navigate by jumping between them.

ElementPurposeOccurrences
<header>Introductory content (logo, nav, hero). Can appear in <article> too.Multiple OK
<main>The primary content unique to this page.One per page
<footer>Closing content (copyright, links). Can appear in <article> too.Multiple OK
<aside>Tangentially related content (sidebar, pull quote).Multiple OK
<nav>Major navigation blocks (main nav, breadcrumbs, pagination).Multiple OK
Q.04 What are the heading hierarchy rules and why do they matter? +

Headings (h1‚Äìh6) create a document outline. Screen readers and search engines use this hierarchy to understand page structure.

Rules

‚Ä¢ One <h1> per page (the main topic)
‚Ä¢ Don't skip levels (h1 ‚Üí h3 is wrong)
‚Ä¢ Choose heading level for meaning, not for visual size (use CSS for sizing)
‚Ä¢ Never use headings just to make text bold/large

<!-- ‚ùå Wrong -->
<h1>Site Name</h1>
<h3>Article Title</h3>  <!-- skipped h2 -->

<!-- ‚úÖ Correct -->
<h1>Site Name</h1>
<h2>Article Title</h2>
  <h3>Section Title</h3>
Q.05 When should you use <figure> and <figcaption>? +

<figure> wraps self-contained content that is referenced from the main flow ‚Äî images, diagrams, code snippets, quotes. <figcaption> provides a caption.

<figure>
  <img src="chart.png" alt="Bar chart showing Q4 revenue growth of 23%">
  <figcaption>Fig. 1 ‚Äî Q4 2024 Revenue Growth by Region</figcaption>
</figure>

The figure could be moved to an appendix without breaking the document flow. That's the semantic test.

Q.06 What is the <picture> element and when would you use it over <img>? +

<picture> lets you serve different images based on screen size or format support. Use it for art direction (different crop on mobile) or format fallbacks (WebP with JPEG fallback).

<picture>
  <!-- Serve AVIF to supporting browsers -->
  <source srcset="hero.avif" type="image/avif">
  <!-- Fall back to WebP -->
  <source srcset="hero.webp" type="image/webp">
  <!-- Art direction: different image on mobile -->
  <source media="(max-width: 768px)" srcset="hero-mobile.jpg">
  <!-- Fallback for all browsers -->
  <img src="hero.jpg" alt="Hero image">
</picture>

ACCESSIBILITY (A11Y)

Building for everyone ‚Äî a must-know for senior roles

Q.01 What is ARIA and when should you use it? +

ARIA (Accessible Rich Internet Applications) is a set of attributes that add semantic meaning to elements where HTML alone is insufficient ‚Äî primarily for custom widgets. The first rule of ARIA: don't use ARIA if native HTML can do it.

<!-- ‚ùå Unnecessary ARIA -->
<button role="button">Click me</button>

<!-- ‚úÖ ARIA where HTML falls short -->
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="dialog-title"
  aria-describedby="dialog-desc"
>
  <h2 id="dialog-title">Confirm Delete</h2>
  <p id="dialog-desc">This action cannot be undone.</p>
</div>
Q.02 What is the alt attribute on images and how should you write good alt text? +

alt describes the image to screen readers and search engines. It shows when the image fails to load. How you write it depends on the image's purpose.

Image Typealt value
Informative (content adds meaning)Describe what the image shows
Decorative (purely visual)alt="" (empty, not omitted)
Functional (button/link image)Describe the action
Complex (chart/graph)Brief description + long description nearby
<img src="chart.png" alt="Bar chart showing sales grew 23% in Q4 2024">
<img src="divider.png" alt=""> <!-- decorative -->
<img src="search-icon.png" alt="Search"> <!-- functional -->
Q.03 What is keyboard accessibility and how do you ensure it? +

All interactive elements must be reachable and operable by keyboard alone (Tab, Shift+Tab, Enter, Space, Arrow keys). Many users with motor disabilities rely entirely on keyboard navigation.

/* ‚ùå Never do this */
*:focus { outline: none; }

/* ‚úÖ Style focus, don't remove it */
*:focus-visible {
  outline: 2px solid #0066ff;
  outline-offset: 2px;
}

/* Make non-interactive elements focusable when needed */
<div role="button" tabindex="0" onkeydown="handleKey(event)">Custom Button</div>
<!-- tabindex="0" adds to tab order; tabindex="-1" allows programmatic focus -->
Q.04 What are WCAG guidelines and what do AA/AAA mean? +

WCAG (Web Content Accessibility Guidelines) are international standards organized around four principles: Perceivable, Operable, Understandable, Robust (POUR).

LevelDescriptionRequirement
AMinimumLegal baseline in many countries
AAStandardMost projects should meet AA
AAAEnhancedStricter, not required everywhere
Key AA Requirements

‚Ä¢ Color contrast ‚â• 4.5:1 for normal text, ‚â• 3:1 for large text
‚Ä¢ All functionality via keyboard
‚Ä¢ No content that flashes >3 times per second
‚Ä¢ Meaningful focus order
‚Ä¢ Error identification and suggestions

Q.05 What is aria-live and when do you use it? +

aria-live tells screen readers to announce dynamic content changes without the user having to navigate to it. Critical for notifications, search results, and form errors.

<div aria-live="polite" aria-atomic="true">
  <!-- Content changes here get announced -->
  <p>3 results found</p>
</div>

<!-- polite: announces after user finishes current action -->
<!-- assertive: interrupts immediately (for errors/alerts) -->
<!-- aria-atomic="true": read entire region, not just changed part -->
Q.06 What is skip navigation and why is it needed? +

A "skip to main content" link that's visually hidden until focused. Lets keyboard users bypass repeated navigation on every page.

<a href="#main" class="skip-link">Skip to main content</a>
<nav>... (long navigation) ...</nav>
<main id="main">...</main>
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  transition: top 0.3s;
}
.skip-link:focus {
  top: 0; /* Revealed on focus */
}

Q.01 What are data-* attributes and when would you use them? +

data-* attributes store custom data on DOM elements. They're accessible via element.dataset in JavaScript. Better than using class names or IDs to store state/configuration.

<button

console.log(btn.dataset.price);       // "19.99"

// Also accessible via CSS

Modern attributes that dramatically improve page loading performance.

Q.05 What is the <template> element? +

<template> holds HTML that won't be rendered on page load but can be cloned and inserted via JavaScript. Its content is inert ‚Äî scripts don't run, images don't load until the template is used.

<template id="card-template">
  <div class="card">
    <h3 class="card-title"></h3>
    <p class="card-body"></p>
  </div>
</template>

const tmpl = document.getElementById('card-template');
const clone = tmpl.content.cloneNode(true);
clone.querySelector('.card-title').textContent = 'Hello';
document.body.appendChild(clone);

Q.05 How does CSS inheritance work? +

Some CSS properties are inherited from parent to children by default. Others are not. You can control this with inherit, initial, unset, and revert.

/* Inherited by default: typography-related */
color, font-family, font-size, font-weight,
line-height, letter-spacing, text-align, visibility

/* NOT inherited: box model */
margin, padding, border, width, height,
display, position, background, overflow

/* Controlling inheritance */
border: inherit;   /* Force inherit from parent */
color: initial;   /* Reset to browser default */
color: unset;     /* inherit if inheritable, else initial */

/* Hex */

/* RGB / RGBA */

Q.07 What are CSS custom properties (variables)? +

CSS custom properties store reusable values. Unlike preprocessor variables (Sass), they're live in the browser ‚Äî they can be changed with JavaScript, scoped to elements, and even animated.

/* Define on :root for global scope */

  --color-primary: #f0c040;
  --spacing-unit: 8px;
  --border-radius: 4px;
}

/* Use with var() */
.button {
  background: var(--color-primary);
  padding: var(--spacing-unit) calc(var(--spacing-unit) * 2);
  border-radius: var(--border-radius);
}

/* Fallback value */
color: var(--color-accent, #ff0000); /* uses red if not defined */

/* Scope to component */
.dark-theme { --color-primary: #1a1a2e; }

/* Update from JavaScript */
document.documentElement.style.setProperty('--color-primary', '#00f');
Q.08 What is the CSS "visually hidden" pattern and when do you use it? +

Visually hidden elements are invisible on screen but remain in the accessibility tree ‚Äî screen readers can still read them. Used for labels, skip links, or additional context for screen readers.

  height: 1px;
  padding: 0;
  margin: -1px;

  white-space: nowrap;
  border: 0;
}

  <span aria-hidden="true">‚úï</span>

BOX MODEL & LAYOUT

Every element is a box ‚Äî understanding it prevents 90% of layout bugs

Q.01 Explain the CSS Box Model. What is box-sizing? +

Every element is a rectangular box composed of four areas: content, padding, border, and margin.

/* content-box (default ‚Äî confusing) */
width: 200px;
padding: 20px;
border: 2px solid;
/* Total rendered width = 200 + 40 + 4 = 244px ‚Üê GOTCHA */

/* border-box (intuitive ‚Äî use this everywhere) */
box-sizing: border-box;
width: 200px;
padding: 20px;
border: 2px solid;
/* Total rendered width = 200px ‚Äî padding+border included ‚úì */

/* Apply globally */
*, *::before, *::after { box-sizing: border-box; }
üí° Interview Tip

Always start your CSS with the box-sizing reset. Explain that border-box makes width predictable ‚Äî what you set is what you get.

Q.02 What is margin collapsing? +

When vertical margins of adjacent block elements meet, they collapse into a single margin equal to the largest of the two. This only happens vertically and only for block-level elements in normal flow.

.box1 { margin-bottom: 30px; }
.box2 { margin-top: 20px; }
/* Gap between box1 and box2 = 30px, not 50px */

/* Margin collapse also happens: */
/* - Parent's top margin collides with first child's top margin */
/* - Empty block's top+bottom margins collapse */

/* Ways to prevent collapse: */
/* - Add padding or border to parent */
/* - Set overflow: hidden on parent */
/* - Use flexbox/grid (collapse doesn't apply) */

FLEXBOX

One-dimensional layout ‚Äî mastering the most-used layout system

Q.01 What is Flexbox and how do you start using it? +

Flexbox is a one-dimensional layout system ‚Äî either a row OR a column. You apply it to a container, and the children (flex items) are arranged in the flex direction.

.container {
  display: flex;             /* Activate flexbox */
  flex-direction: row;        /* row | row-reverse | column | column-reverse */
  flex-wrap: nowrap;          /* nowrap | wrap | wrap-reverse */
  
  /* Shorthand: flex-flow = direction + wrap */
  flex-flow: row wrap;
}
Main Axis vs Cross Axis

The main axis follows flex-direction. The cross axis is perpendicular. justify-content controls main axis; align-items controls cross axis.

Q.02 Explain justify-content, align-items, and align-content. + 
/* justify-content ‚Äî main axis alignment */

justify-content: flex-end;
justify-content: center;
justify-content: space-between;  /* items at edges, space between */
justify-content: space-around;   /* equal space around each item */
justify-content: space-evenly;   /* equal space between all gaps */

/* align-items ‚Äî cross axis (single line) */
align-items: stretch;     /* default: fill cross axis */
align-items: flex-start;
align-items: flex-end;
align-items: center;      /* vertical centering! */
align-items: baseline;    /* text baseline alignment */

/* align-content ‚Äî cross axis (multi-line, only with flex-wrap) */
align-content: flex-start | flex-end | center | space-between | stretch;
Q.03 What do flex-grow, flex-shrink, and flex-basis do? +

These three properties (combined as the flex shorthand) control how flex items size themselves relative to available space.

flex-growHow much extra space an item takes (ratio)0 (don't grow)
flex-shrinkHow much an item shrinks when space is tight1 (can shrink)

Q.04 How do you center an element perfectly with Flexbox? +

  align-items: center;      /* vertical */
  min-height: 100vh;        /* needs height to center vertically */

.nav-links .login { margin-left: auto; } /* push to far right */

2. Forgot to set a threshold (top, bottom, etc.) ‚Äî required!
3. Element is already as tall as its parent ‚Äî no room to stick
4. Parent uses display: flex with align-items: stretch (stretch = full height)

Q.01 What is mobile-first design and why is it preferred? +

Mobile-first means writing base CSS for small screens, then using min-width media queries to enhance for larger screens. It's preferred because it forces prioritization of content, results in less CSS on mobile (better performance), and aligns with how most users access the web.

/* ‚ùå Desktop-first (overriding with max-width) */
.sidebar { width: 300px; }
@media (max-width: 768px) { .sidebar { display: none; } }

/* ‚úÖ Mobile-first (enhancing with min-width) */
.sidebar { display: none; }
@media (min-width: 768px) { .sidebar { display: block; width: 300px; } }

SPECIFICITY & THE CASCADE

Why your styles aren't applying ‚Äî and how to master the rules

Q.01 How is CSS specificity calculated? +

Specificity is a 4-column score: (inline, ID, class/attr/pseudo-class, element/pseudo-element). Higher specificity wins. Equal specificity ‚Üí last one in source wins.

SelectorScore
* { }0,0,0,0
p { }0,0,0,1
p::before { }0,0,0,2
.class { }0,0,1,0
[type="text"] { }0,0,1,0
p.class { }0,0,1,1
#id { }0,1,0,0
style=""1,0,0,0
!importantOverrides all (except other !important)
/* :is() and :not() take the specificity of their argument */
:is(h1, #id)  /* specificity of #id = 0,1,0,0 ‚Äî highest arg wins */
:where(h1, h2)  /* always 0 specificity ‚Äî great for base styles */
Q.02 What is the CSS cascade and what factors determine which styles win? +

The cascade resolves conflicts between declarations. Order of priority (highest first):

Cascade Priority Order

1. Origin & Importance: Transitions > !important user-agent > !important author > !important user > animations > author > user > user-agent
2. Specificity: Higher wins
3. Order: Later in source wins (when specificity ties)
4. Inheritance: From parent (when no matching rule)

Q.03 When should (and shouldn't) you use !important? +

!important breaks the natural specificity flow. It's usually a sign of poorly structured CSS.

Legitimate uses

‚Ä¢ Utility classes that must always apply (e.g., .hidden { display: none !important; })
‚Ä¢ Overriding third-party library CSS you can't control
‚Ä¢ User stylesheets for accessibility overrides

üí° Interview Tip

If you're using !important in your own codebase, it's a code smell. Better to refactor the CSS architecture ‚Äî use lower specificity, add a modifier class, or use CSS custom properties.

Q.04 What are CSS Layers (@layer) and why are they useful? +

@layer lets you define explicit cascade layers, giving you full control over which styles win regardless of specificity.

/* Declare layer order ‚Äî later layers win */
@layer reset, base, components, utilities;

@layer reset {
  * { margin: 0; padding: 0; }
}

@layer components {
  .button { background: blue; }
}

@layer utilities {
  .text-red { color: red; }  /* always wins over components */
}

/* Styles outside layers always win over layered styles */
Q.05 What is the :is(), :where(), and :has() pseudo-class? + 
/* :is() ‚Äî match any of the selectors (takes highest specificity) */
:is(h1, h2, h3) a { color: blue; }
/* Same as: h1 a, h2 a, h3 a */

/* :where() ‚Äî same but always zero specificity */
:where(h1, h2, h3) a { color: blue; }
/* Easy to override ‚Äî great for base/reset styles */

/* :has() ‚Äî parent selector! */
figure:has(figcaption) { border: 1px solid; }
input:has(+ .error-msg) { border-color: red; }
.card:has(:hover) { box-shadow: 0 4px 20px rgba(0,0,0,.2); }
form:has(input:invalid) { --border: red; }

Q.01 What is the critical rendering path? +

The sequence of steps the browser takes to convert HTML, CSS, and JS into pixels on screen.

Steps

1. Parse HTML ‚Üí DOM (Document Object Model)
2. Parse CSS ‚Üí CSSOM (CSS Object Model)
3. Combine ‚Üí Render Tree (only visible elements)
4. Layout (Reflow) ‚Üí calculate size and position of each element
5. Paint ‚Üí fill in pixels (colors, shadows, text)
6. Composite ‚Üí combine layers, send to screen

CSS is render-blocking ‚Äî the browser won't show content until it has processed all CSS. JS in <head> without defer/async is also render-blocking.

Q.02 What triggers reflow (layout) and how do you avoid it? +

Reflow recalculates the geometry of every affected element ‚Äî expensive. Reading layout properties forces a synchronous reflow (called "layout thrashing").

// ‚ùå Layout thrashing ‚Äî forced reflow on every iteration
elements.forEach(el => {
  el.style.width = el.offsetWidth + 10 + 'px';  // read ‚Üí write ‚Üí read...
});

// ‚úÖ Batch reads first, then writes
const widths = elements.map(el => el.offsetWidth);  // all reads
elements.forEach((el, i) => {
  el.style.width = widths[i] + 10 + 'px';           // all writes
});

// ‚úÖ Even better: use CSS transforms instead of geometry
el.style.transform = 'translateX(10px)'; // compositor only, no reflow
Q.03 What is critical CSS and how do you implement it? +

Critical CSS is the minimal set of styles needed to render above-the-fold content. Inlining it in <head> eliminates the CSS download that blocks first paint.

<head>
  <!-- Inline critical CSS -->
  <style>
    /* minimal styles for header, hero, and first screen */
    body { margin: 0; font-family: sans-serif; }
    .header { ... }
    .hero { ... }
  </style>
  
  <!-- Defer non-critical CSS -->
  <link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
  <noscript><link rel="stylesheet" href="styles.css"></noscript>
</head>
Q.04 What is content-visibility and contain? + 
/* content-visibility: skip rendering off-screen sections */
.article {
  content-visibility: auto;         /* browser skips layout/paint until near viewport */
  contain-intrinsic-size: 0 500px; /* placeholder size to prevent scroll jump */
}

/* contain: tell browser element is independent */
.widget {
  contain: strict;    /* layout + style + size + paint */
  contain: layout;    /* changes inside don't affect outside layout */
  contain: paint;     /* children don't render outside bounds */
}
Q.05 What are CSS best practices for maintainability? +
Best Practices

‚Ä¢ Use box-sizing: border-box globally
‚Ä¢ Prefer class selectors ‚Äî keep specificity low
‚Ä¢ Avoid deeply nested selectors (max 3 levels)
‚Ä¢ Use CSS custom properties for design tokens
‚Ä¢ Write mobile-first with min-width queries
‚Ä¢ Prefer transform/opacity for animation
‚Ä¢ Use :focus-visible not :focus for keyboard styles
‚Ä¢ Add prefers-reduced-motion support
‚Ä¢ Use semantic HTML over div-soup + ARIA
‚Ä¢ Use logical properties for i18n support
‚Ä¢ Avoid !important except for utilities
‚Ä¢ Use CSS @layer for architecture
